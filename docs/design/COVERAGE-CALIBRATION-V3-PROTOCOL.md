## 0. What this measures, and the (refined) safety asymmetry — plain language

A space-filling sampler digs across a grid of strategy settings; each grid square should be **one meaningfully-different strategy**. A square's width = the **just-noticeable-difference (JND)**: the smallest change to a dial (e.g. SMA period 20→22) that *actually changes what the strategy does* on real prices. The count of distinct squares later becomes **N**, the multiple-testing correction in the deflated-Sharpe (DSR) gate.

**The danger is asymmetric — but there are TWO dangers, not one.** The draft treated "finer = unboundedly safe." The bias hunt proved that is false at the extremes, so the FINAL rule is sharper:

- **PRIMARY danger — too coarse.** Merging genuinely-different strategies into one square **undercounts N** → the DSR hurdle is too low → the engine calls noise "an edge." This is the **false-discovery / unsafe** direction. **Break every *genuine* tie toward finer.**
- **SECONDARY danger — artifactually too fine.** A grid driven *finer by measurement artifacts* (a single pathological name, a strategy killed to zero trades scored as "maximally distinct," a one-bar flip at the quantization floor, or the minimum of several noisy estimates) **overcounts N** → because `sr0 ≈ √(2·ln N)` the hurdle eventually rejects *everything* (vacuous gate) → and, worse, vacuity tempts a *post-hoc coarsening* of N, which is the unsafe direction by the back door.

**FINAL discipline:** keep *genuine-signal* fineness (Q<1 over assets, bootstrap-separated min over bases/regimes, strict regime-min) — but **strip *artifactual* fineness with pre-registered quality gates BEFORE it can set the grid** (saturation exclusion, an active-bar quantum floor, single-asset robustness). Residual vacuity is handled only by a **fully-frozen effective-N reduction** (§7), never by coarsening the grid after seeing a result. So a measurement *error* still errs toward finer/larger-N; but the grid is not hostage to non-economic artifacts.

**The metric** (shipped `positions_of()`/`disagreement()`): for a `(template, params, asset)` run the *real* backtest, reconstruct the discrete held-position vector (long/flat/short) per bar from the trades, and compare two settings by the **fraction of bars where they hold a different state, normalised by bars where at least one is in-market** (union-in-market). Return-magnitude disagreement was rejected (saturates ~1.0 on share-count/entry-price noise). Determinism is verified 0.0. **JND** = smallest step at which this disagreement first crosses target **T** (primary **T = 0.05**).

**Scope honesty about N (the category caveat, §7).** Position-state distinctness answers *"is this a different strategy to sample?"* The DSR N assumes *~independent trials*. These are not the same object: two settings that agree on 95% of in-market days have ~0.9+ correlated return streams; and union-in-market normalization *discards* flat-day agreement, so a low-exposure pair can look "distinct" while its full return series correlate 0.95. Therefore the JND grid **sizes the sampler** (its shipped, correct purpose, unchanged), and for N the calibration run **additionally measures** the adjacent-cell full-series return correlation ρ̄ so the effective-N reduction is *empirical, not guessed* (§7). This is the honest bridge from JND to N.

**[FIX-13] Instrument the swallowed exception — CONFIRMED.** The bare `except Exception: p = np.zeros(0)` (`:91`) makes any engine failure indistinguishable from a legitimate guard-skip, and on 198 gappy/penny names it will fire non-randomly per (dial,name) = uncontrolled censoring. **FIX:** narrow the except to expected engine errors; **record** `(dial,base,step,asset,exception_type,skip_reason)` into a parallel **valid/skip mask** committed with the tensor; **fail the whole run loudly** on any non-enumerated exception class. Distinguish three outcomes explicitly — **crashed** (empty), **legitimately-flat** (all-zeros, len>0), **traded** — so §3.2/§3.3 can use them.

**[FIX-14] Degenerate-saturation classification (high→blocker at scale).** CONFIRMED: a step that drives a strategy to **zero trades** yields an all-flat vector; against a trading base, `active = base's in-market bars` and every one differs → `disagreement = exactly 1.0`. So "dead" is scored "maximally distinct," and Q=0.85 preferentially selects these ≈1.0 readings → JND collapses to the smallest step → N inflates toward vacuity (this, not "one pathological name," is the real vacuity mechanism). **FIX:** before aggregating, classify each δ reading **live-vs-live vs degenerate** — a transition where *either* side has `< 3 round-trips` or exposure_time below a floor is **degenerate on/off**, not "meaningfully different." Pre-registered handling: **exclude degenerate transitions from the Q aggregation** (measure distinctness among live-vs-live pairs only); **report** per (dial,base,step) the fraction of the Q=0.85 mass contributed by ≈1.0 saturated readings; **fail pre-registration** if any dial's JND is set by a step whose `D_0.85` is >X% saturated.

**[FIX-15] Active-bar quantum floor — raise the guard, add an absolute-bar minimum.** CONFIRMED: at the 20-active-bar guard the metric's quantum is `1/20 = 0.05 = T`, so a *single* flipped bar equals T = pure noise, and Q=0.85 selects exactly these thin names. **FIX:** replace `<20 active` with **`active bars ≥ N_min = ceil(c/T)` with c=5 ⇒ ≥100 active bars for T=0.05** (quantum ≤0.01, 5× margin), applied as **dial-specific eligibility** (a name is valid for a (dial,step) only if *both* settings have ≥ N_min active bars). Additionally require a **minimum absolute differing-bar count ≥ 5** so a 1–2 bar flip can never cross T. Report the active-bar distribution per (dial,regime).

**[FIX-16] Short-history × long-period × warmup contamination.** Delisted names admit down to 250 bars, but bases reach slow_period=200; with `warmup_bars=0` an SMA(150) is invalid for ~150 bars and the runner ffills. **FIX:** dial-specific min-bars — a name is eligible for a period dial only if it has `≥ k·max(period)` bars with `k≥3` (i.e. ≥450 bars for a 150-base), *and* re-run the C4 warmup check **on the short-history delisted subset specifically**; if Δ is material there, discard `max(period)` warmup bars in `positions_of` for calibration.

**[FIX-17] Truncation / death-spiral / stale-tail trim for delisted names.** `finalize_trades=True` (`runner.py:279`) force-closes the terminal trade **at the delisting bar**, so a death-spiral's extreme vol inflates flips right before delisting (a removal artifact, not market behaviour), and EODHD sometimes carries a stale constant-`adjc` tail that deflates disagreement. **FIX (frozen):** for delisted names, (a) drop the final **K=5–10** trading days before `last_date` from the position vector, and (b) drop any **trailing run of ≥3 constant-`adjc` (zero-return) bars**; log trimmed-bar counts per name in the committed results. (Note: the `finalize_trades=True` path means the Lens-5 "NaT terminal position" concern is largely moot — exit_time is populated; the only residual is the symmetric single-final-bar `< x` exclusion, handled by treating a force-closed terminal trade's exit as inclusive.)

**[FIX-18] Sustained-crossing JND + interpolation (replaces first-crossing).** The shipped JND is literal *first*-crossing (`:139`), which (a) latches onto a transient bump if δ(step) is non-monotone, and (b) rounds the crossing **up** to the next sampled step = the one place the pipeline rounds *coarse*. **FIX:** (i) pre-register a **monotonicity diagnostic** (fraction of assets whose δ(step) is non-monotone beyond tolerance) and report it; if material, define JND as the **sustained crossing** — smallest `s` with `D_Q(s')≥T` for **all** `s'≥s` (last up-crossing), robust to dips; (ii) **linearly interpolate** `D_Q` between the bracketing steps to locate the crossing rather than rounding up. State that the finer-is-safe direction is proven only under the verified monotonicity premise.

**[FIX-19] Extend-finer rule (symmetric to the integer handling).** If any dial's measured JND lands on the **finest rung** of its sweep (0.05 ratio / 1 pt / 0.1 mult) for **any** base or regime, the grid is too coarse to bound it. Deterministic pre-committed response: prepend finer rungs (`RATIOS` +[0.02,0.03]; threshold +[0.5]; mult +[0.05]) and re-measure that dial until the JND is strictly *interior* to the sweep. "My step grid bottomed out" can never be accepted as fine-enough.

**The one expensive artifact** is the raw per-asset disagreement tensor `δ(dial, base, step, asset, regime)` + its valid/skip mask + exposure + adjacent-cell return-correlation streams (§7). Compute once, commit (`calibration-v3-results.json`). Every aggregation/quantile/regime/bootstrap decision is then a cheap, re-derivable array transform.

## 4. Conservative cross-asset aggregation (the C3 fix), guarded against artifacts

The v2 defect: **median** disagreement across assets, then **median** JND across bases — median ignores the fine-JND high-vol tail = leniency = unsafe. The object aggregated is per-step **DISAGREEMENT (δ)**, not per-asset JND (a max-of-JND would go *coarser* = wrong direction).

**Equivalence (correctness proof, verified).** Under per-asset monotonicity of δ(step), taking the per-step Q-quantile across assets then first-crossing T is **exactly** reading the `(1−Q)` percentile of the per-asset JND distribution: `D_Q(s)≥T ⟺ ≥(1−Q) of assets have δ(a,s)≥T ⟺ ≥(1−Q) have JND_a≤s`. So Q=0.85 → grid width = the 15th-percentile-smallest JND: finer than 85% of names need, coarser than the top 15%. Higher Q → smaller crossing step → finer JND → more cells → larger N. Direction verified. (This equivalence is why the monotonicity diagnostic [FIX-18] is load-bearing, and why **T and Q are one conservatism knob, not two** — see below.)

**Frozen aggregation pipeline:**
1. `δ(d,b,s,a,regime)` = shipped metric, with the FIX-13/14/15/17 masks applied (degenerate and sub-N_min readings excluded, not read as signal).
2. **Cross-asset:** `D_Q(d,b,s,regime) = Q-th percentile of {valid, live-vs-live δ}`, **Q = 0.85**; require **≥ M_min = 30 valid census-balanced assets** else undefined. (Q<1 is the single-asset-max guard: `max` is hostage to one pathological name → grid absurdly fine → gate vacuous.)
3. **Per-base JND:** smallest interpolated/sustained `s` with `D_Q ≥ T`.
4. **[FIX-20] Cross-base: bootstrap-separated min, not raw min (winner's-curse fix, high).** The draft took the raw MIN over 3 noisy bases (and conditional min over 6 regimes) = effectively min over ~18 noisy tail-quantiles → biased **fine** by ~0.85σ (k=3) to ~1.8σ (k=18), un-budgeted, and *multiplicative* across dials into N. **FIX:** a base (or regime, §5) may only drive the dial **finer** if its JND's **bootstrap upper-CI is below the pooled runner-up's point estimate** (statistically separated). Bases whose CIs overlap are **pooled** (one δ distribution → better quantile). Report **raw-min vs CI-separated-min** per dial so the selection bias is visible and bounded. The frozen resolution is the CI-separated min (genuine fineness kept; noise-driven fineness rejected). All-`None` ⇒ dial inert ⇒ `_KIND_FALLBACK` (recomputed) + flag.

**Why Q = 0.85 (pre-registered, justified).** `max` → vacuity via one glitch; `median` = the unsafe defect. 0.85 sits on the fine/safe side of the tail: only the top ~15% of names have a finer true JND than the grid (bounded residual), while a ~15% buffer means 1–29 outliers cannot set it. **[FIX-21] The full ladder {0.50, 0.75, 0.85, 0.95, max} × {T 0.02,0.05,0.10} is REPORTED for transparency ONLY; the single (T=0.05, Q=0.85) cell is THE frozen estimator and the ladders can never re-select the resolution** (closes the forking-paths menu — T and Q push the same way per the equivalence above).

**Safety direction (explicit, refined).** Q<1 over assets + CI-separated-min over bases + strict regime-min (§5) all move *genuine* resolution finer → more cells → larger N → higher `sr0` → harder to pass → fewer false discoveries. The **only** way anything moves coarse is (i) the pooled runner-up governing when a finer candidate is not bootstrap-separated (a *noise* rejection, not a signal rejection), or (ii) the fully-frozen effective-N reduction (§7). Nothing moves coarse after seeing a result.

## 5. Regime handling — strict finest-across-regimes, quality-gated (the temporal-axis fix)

The draft made the **pooled full-window** the PRIMARY JND with regimes demoted to a `>1-step` hysteresis check. **[FIX-22, high] This is the wrong default.** The shipped `disagreement()` normalizes crisis-driven flips by union-in-market over the **entire ~6000-bar** window, so a ~250–750-bar crisis is diluted by 20 calm years; the 85th-percentile is *across assets* and cannot recover *within-asset temporal* dilution. Applying the protocol's own "coarse is the only unsafe direction, break ties finer" rule to the **temporal axis**:

**PRIMARY JND = strict min-across-regimes:** compute δ per-regime and set `JND(d) = ` the finest bootstrap-separated crossing over **{full window} ∪ {5 regimes}** (via the §4-step-4 CI-separated-min, now over the regime axis too). **[FIX-23] The `>1-step` hysteresis deadband is DELETED** — it was a silent coarse-direction leniency contradicting §0. A finer regime wins by any *statistically-separated* margin.

**[FIX-24] Vacuity is prevented by per-regime QUALITY GATES, not by the deadband and not by keeping the diluted pooled measurement.** A regime may only *set* (be the min for) a dial if, in that regime: (a) valid-asset count ≥ M_min **and** census-balanced; (b) the crossing step's `D_Q` is **non-saturated** (< the degenerate ceiling, FIX-14); (c) active bars satisfy N_min so T is above quantum (FIX-15); (d) its JND's bootstrap **upper-CI is below** the full-window point estimate (FIX-20 separation). A regime that fails any gate is **reported as "finer-but-unreliable," never adopted.** This is the synthesis of the strict-min (survivorship/regime lens) and the "regime-min imports noise" (metric/aggregation lenses): the finest *reliable* regime governs; a thin/noisy COVID blip near M_min cannot vacuum the grid.

## 6. Stability / sample-size rule — turning "≥150" into evidence

**Bootstrap (frozen, B=500, seed 20230706):** resample the **asset axis of the committed δ-tensor** with replacement, re-run the entire §4–§5 aggregation per resample → a bootstrap distribution of `JND(d)`. No backtests re-run (pure array resampling → cheap). **[FIX-26] The bootstrap also resamples and reports the valid-count and saturated-fraction distributions per (dial,regime), and the convergence criterion is evaluated on the non-saturated, census-balanced subset** — the bootstrap *propagates* the guard-censoring (FIX-13/14/15), it cannot correct it, so "stable" must mean "stable on the clean subset," not "stable on a consistently-biased estimator."

**Convergence (frozen, asymmetric — finer is safe):** for each tradeable dial, bootstrapped `JND(d)` lands on the point-estimate step **or the adjacent finer step in ≥90%** of resamples; coarser-by-≥1-step in >10% = **instability flag**. Tolerance = one grid step (±1 for integer dials).

**[FIX-10] Effective-tail reporting.** Because Q=0.85's effective sample is the top ~15% of names, report the **effective tail count** (names with vol ≥ p75) and *its* bootstrap CI, not just n≥150. The panel is sized to the tail (≥50 tail names, §2.4), so the bootstrap does not routinely drop the whole tail.

**Subsample ladder (frozen — makes ≥150 a curve):** re-measure headline `JND(d)` on strata-preserving seeded subpanels {50, 75, 100, 125, 150, full}, plotting JND vs n **on the *effective* n per (dial,regime) cell**, not nominal panel size. **Stopping rule:** stop when, for every tradeable dial, (a) resolution unchanged or only finer-by-≤1-step between the 125- and 150-subpanels, **and** (b) the ≥90% bootstrap criterion is met at 150, **and** (c) the effective-tail count ≥ its floor. **On failure (pre-committed):** enlarge the dial's sensitive strata (add high-vol small-cap / delisted names — the finest-JND tail) and re-bootstrap, OR adopt the finest step inside the bootstrap 90% CI. Deterministic response, not a judgement call. Cells with effective n below a pre-registered floor are marked low-confidence and **barred from setting** the frozen resolution (report only).

## 7. From JND to the DSR N — the honest bridge (metric-category fix)

This section is what makes the frozen resolutions *honestly* usable as N. It resolves the metric-category blocker and pins the parts of B3/B5 that depend on the measurement (the code wiring stays in `COVERAGE-MEMORY-V2-PLAN.md`).

**[FIX-27, blocker] Measure the correlation the reduction needs — do not guess it.** The JND grid answers "different strategy to sample?"; the DSR N assumes "independent trial." The calibration run **additionally emits, from the same backtests**, the **adjacent-cell full-series return correlation** (Pearson of the two settings' per-bar return streams — *not* union-in-market position disagreement, *not* the rejected return-magnitude metric). From this:
- **ρ̄** = mean adjacent-cell full-series return correlation per template, measured on the panel (frozen estimator, committed).
- **Effective N** = the F1 `reachable_cells` count (the conservative upper bound, B5) reduced via a **pre-registered** Galwey / `1 + (N−1)·ρ̄` mapping using the **measured** ρ̄ — never a guessed reduction, never inflated past the raw count.
- **Self-consistency gate:** verify empirically that T=0.05 position-disagreement corresponds to **ρ ≳ 0.9** on the diverse/fat-tailed panel. If 5% position-disagreement lands on high-return days (high-vol names) and yields ρ≈0.5, the "adjacent cells ~independent-ish upper bound" story is **falsified** and N must be built from **return-decorrelation clustering** (pre-registered fallback), not JND cells. Report the empirical position-disagreement↔ρ map.

**[FIX-28] Co-scope V with N (B3):** the gate's `sr0 = √V·f(N)` requires V from the **same null population** as N. Use the **theoretical null V ≈ 1/T_bars** (variance of a zero-skill per-period Sharpe over the return length) — needs no stored performance, preserving the overfitting-neutral firewall — never per-run V paired with a grid N. Pre-register this; do not let the gate run on the defaulted `trial_sr_variance=0.001` (`deflated_sharpe.py:84`) when N is grid-sized.

**[FIX-29] Split `n_trials` (B4):** a realized-executed-trials counter drives the provisional/auto-pass valve (`<2` auto-pass, `<20` provisional); a separate `search_size` = the effective N enters `sr0` only. (Wiring task; noted so the calibration output is shaped to feed both.)

**[FIX-30] `sr0(N)` sensitivity pre-check (before freezing).** Because `sr0 ≈ √(2·ln N)` the median→Q=0.85 change moves the hurdle by a whisker, while the saturation failure (FIX-14) can move N by orders of magnitude and tip the gate vacuous. Compute and **commit the `sr0(N)` curve** across `{median-N, Q0.85-N, saturated-N}` at the theoretical V, showing where the DSR pass/fail boundary actually moves. This quantitatively justifies prioritizing the saturation/quantum guards over the quantile choice, and it gates the whole N-refinement behind V being **measured** (not defaulted).

**[FIX-31] Integer-vs-ratio: analytic rule PRIMARY (prevents N explosion).** The draft re-runs the integer/ratio classifier with the +1-int **85th-pct** across 165 names; on a high-vol panel that quantile can exceed T for sma/macd too, flipping them to one-cell-per-integer → a period dim jumps from ~10–15 log cells to 45–180 integer cells → N explodes. **FIX:** classify integer-governed **iff `base×r < 1` integer** (a *structural* property, robust to the quantile — already the analytic cross-check in `validate_grid_noise.py:12`). The empirical `+1-int D_0.85 ≥ T` is **confirmatory only**. Pre-committed response if sma/macd empirically flip: keep the ratio grid and reflect the higher per-integer sensitivity via the effective-N ρ̄ term (§7), **not** by granting every integer its own independent cell.
